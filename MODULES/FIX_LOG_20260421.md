# Fix Log — 2026-04-21

## Priority 1: RLS SELECT Policies for Participants

**Problem:** 14 tables had moderator-only SELECT policies. Participants using the anon key got empty results from most queries. This was the root cause of most "loading/empty" issues.

**Migration: `add_participant_select_policies`**

Added `FOR SELECT USING (true)` policies to:
- sim_runs, roles, role_actions — core game state
- sanctions, tariffs — economic data
- world_state — global state
- org_memberships, organizations — org data
- role_relationships — personal relationships
- zones — map data

Added full CRUD policies to tables with NO policies at all:
- blockades, observatory_combat_results, power_assignments, basing_rights

## Priority 1b: Realtime Publication

**Problem:** Only 16 of 65 tables were published for Supabase Realtime. Changes to roles, sanctions, tariffs, etc. required page refresh.

**Migration: `publish_missing_realtime_tables_v2`**

Added 10 tables to supabase_realtime publication:
- roles, role_actions, sanctions, tariffs, blockades
- org_memberships, world_state, basing_rights
- power_assignments, observatory_combat_results

**Total published: 26 tables** (was 16)

## Impact

Participants should now be able to:
- Load their role, actions, and country data without moderator auth
- See sanctions/tariffs changes in real-time
- See role status changes (arrest, killed) live
- See org membership changes live
